OAuth 2.0 — Google Login
OAuth 2.0 lets users sign in with an existing account (Google, GitHub) without creating a new password. Your app never sees the user's Google password.
OAuth 2.0 Authorization Code Flow
1. User clicks "Login with Google"
   → Redirect to accounts.google.com/oauth?client_id=...&redirect_uri=...
2. User logs in on Google's page (your app sees NOTHING)
3. Google redirects back with ?code=AUTH_CODE
4. Your backend: POST /token → exchange code for access_token
5. Backend fetches user profile from Google API
6. Backend issues your own JWT → sends to frontend
7. Frontend stores JWT, uses it for all future requests
Spring Boot OAuth2 Setup (pom.xml)
<dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-oauth2-client</artifactId> </dependency> # application.yml spring.security.oauth2.client.registration.google: client-id: YOUR_GOOGLE_CLIENT_ID client-secret: YOUR_SECRET scope: email, profile

CORS — Cross-Origin Resource Sharing
This HTML page runs on a different origin than the Spring backend (localhost:8080). Without CORS headers, the browser blocks all requests.
How CORS Works
Browser (file:// or localhost:3000)
  → OPTIONS /api/polls (preflight)
Spring Security (localhost:8080)
  → 200 + Access-Control-Allow-Origin: *
Browser: ✓ CORS allowed, send real request
  → GET /api/polls (with JWT)
SecurityConfig CORS Bean
config.setAllowedOrigins(List.of( "http://localhost:3000", // React "http://localhost:5173", // Vite "http://127.0.0.1:5500", // VS Code Live Server "null" // file:// origin )); config.setAllowedMethods(List.of("GET","POST","PUT","DELETE","OPTIONS")); config.setAllowedHeaders(List.of("*")); config.setAllowCredentials(true);
